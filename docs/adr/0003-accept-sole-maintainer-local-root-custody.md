# Accept sole-maintainer local custody for the initial production root

For the initial production signing setup, one maintainer may generate and retain the root and delegated private keys on the FileVault-protected maintainer workstation instead of conducting a witnessed offline ceremony. Private keys remain human-custodied: no agent may generate, read, receive, or configure them; the root private key never enters GitHub; and only the delegated key enters the protected `production-release` Environment. This deliberately trades offline isolation and independent witnesses for an operable sole-maintainer process. Suspected workstation compromise is therefore suspected root compromise and requires the reviewed re-bootstrap path; moving to stronger offline or hardware custody later requires a reviewed new-root transition rather than assuming old copies are gone.
